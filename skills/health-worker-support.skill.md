# Skill: Health Worker Support

## Skill Name
`health-worker-support`

## Purpose
Generate a concise, structured patient briefing note for community health workers (ASHA, ANM, CHW) who will follow up with the patient in person. The briefing summarizes the triage outcome, key symptoms, recommended actions, and relevant government health schemes, enabling health workers to provide targeted, informed care.

## When to Use
- Called after Follow-up & Adherence Agent has generated the follow-up plan.
- Use when implementing the `health_worker_support_agent` function.
- Use when an ASHA/ANM/CHW needs a summary before a home visit.
- Use when building the health worker dashboard module.
- Use when designing the briefing note format and content.

## Inputs Expected

```json
{
  "session_id": "uuid",
  "patient_token": "PT-xxxxxxxx",
  "timestamp": "2026-06-27T10:30:00Z",
  "chief_complaint": "Fever with headache for 3 days",
  "symptoms": ["fever", "headache", "body aches", "chills"],
  "symptom_duration": "3 days",
  "symptom_severity": "moderate",
  "triage_level": "MODERATE",
  "triage_reasoning": "Persistent fever with headache needs PHC evaluation",
  "health_guidance": "RAG-grounded guidance text",
  "rag_sources": ["WHO_Fever_Guidelines_2023"],
  "recommended_facility": "Dharmapuri PHC",
  "followup_plan": {...},
  "language": "hi"
}
```

## Output Format

```json
{
  "health_worker_briefing": "Full structured briefing note text",
  "briefing_format": "text",
  "key_actions": ["Visit patient within 24 hours", "Check temperature", "Refer to PHC if not improved"],
  "scheme_eligibility": ["NHM Free Drug Scheme", "Ayushman Bharat"]
}
```

## Decision Rules

### Briefing Note Structure
```
PATIENT BRIEFING NOTE — RuralCare AI
=====================================
Session : {session_id[:8]}...
Token   : {patient_token}
Date    : {formatted_date}
Language: {language}

TRIAGE SUMMARY
--------------
Level    : {triage_level}
Complaint: {chief_complaint}
Duration : {symptom_duration}
Severity : {symptom_severity}
Symptoms : {comma-separated symptoms}
Reasoning: {triage_reasoning}

HEALTH WORKER ACTION REQUIRED
------------------------------
{action_instructions based on triage_level}

KEY CLINICAL NOTES (RAG-Grounded)
----------------------------------
{2-3 sentence summary of health_guidance for health worker context}
Source: {rag_sources}

FOLLOW-UP SCHEDULE
------------------
Visit patient: {followup_plan.follow_up_in}
Monitor for : {followup_plan.watch_for}
Escalate if : {followup_plan.return_immediately_if}

GOVERNMENT SCHEME ELIGIBILITY
------------------------------
{scheme_eligibility list}

NOTES FOR HEALTH WORKER
------------------------
- This summary was generated by RuralCare AI from patient-reported symptoms.
- Clinical judgment of the health worker overrides this briefing.
- Refer to a doctor for diagnosis and treatment decisions.

[Standard Disclaimer]
```

### Action Instructions by Triage Level
```
EMERGENCY:
  "URGENT: This patient needs emergency care. Arrange transport to the nearest
   hospital immediately. Call 108 for ambulance if available. Stay with patient."

URGENT:
  "Visit patient within 2-4 hours. Assess temperature, breathing, hydration.
   Arrange referral to CHC or district hospital if condition is serious."

MODERATE:
  "Visit patient within 24 hours. Check temperature and hydration.
   Ensure patient has a plan to visit the PHC within 48 hours."

MILD:
  "Follow up with patient in 3 days. Provide general health guidance.
   Remind patient to seek care if symptoms worsen."
```

### Scheme Eligibility Logic
```python
schemes = []
if is_government_facility_recommended:
    schemes.append("NHM Free Drug Scheme — free medicines at PHC/CHC")
if triage_level in ["URGENT", "EMERGENCY"]:
    schemes.append("PM-JAY / Ayushman Bharat — covers hospitalization")
if "pregnant" in symptoms or "maternal" in chief_complaint.lower():
    schemes.append("JSSK — free care for pregnant women and newborns")
if "child" in chief_complaint.lower() or "infant" in chief_complaint.lower():
    schemes.append("RBSK — Rashtriya Bal Swasthya Karyakram for children")
if "anemia" in symptoms or "iron" in symptoms:
    schemes.append("National Iron Plus Initiative — free iron supplements")
```

### Language Note for Health Worker
If patient's language != "en":
- Add note: "Patient's preferred language: {language_name}. Communicate in that language during home visit."

## Safety Rules
- Briefing note uses clinical language appropriate for trained health workers.
- Still does NOT suggest a specific diagnosis — health worker makes clinical judgment.
- PHI (real name, phone, address) must not appear in briefing — patient token only.
- Briefing note must include "Clinical judgment of the health worker overrides this briefing."
- Emergency briefings must show escalation action FIRST, before any other content.

## Failure Handling
- **LLM fails:** Generate briefing from template using state data only — no LLM required for structured briefing.
- **Missing triage_level:** Default to URGENT; note "Triage level unavailable — treat as urgent."
- **Empty symptoms:** Include "Symptom data unavailable — conduct full assessment during home visit."
- **Language mismatch:** Always generate briefing in English (health workers use English protocol documents), with language note for patient communication.
